Credit Card Fraud Analysis | SQL + Power BI Project

Project Overview:
This project focuses on analyzing credit card transaction data to identify fraud patterns, risky transaction behaviors, and high-risk customer segments using SQL and Power BI.

The project includes:
Data Cleaning using SQL
Fraud Pattern Analysis
KPI Development
Interactive Power BI Dashboard
Business Insights & Recommendations

Tools Used:
SQL (MySQL)
Power BI
Excel

Dataset Features:
The dataset contains
Transaction Date & Time
Transaction Amount
Transaction Type
Card Type
Merchant Information
Merchant Location
Fraud Status
Error Details

Data Cleaning Performed:
Handled missing values
Fixed incorrect data types
Converted amount column into numeric format
Created time period categories
Created amount range categories
Removed inconsistencies
Prepared clean dataset for Power BI

Key Business Questions:
Which transaction method has the highest fraud risk?
Which card types are most fraud-prone?
During which time period does fraud occur most?
Which months show the highest fraud activity?
Are high-value transactions riskier?
Which merchant locations have higher fraud concentration?

Key Insights:

1. Online Transactions Have Highest Fraud Rate
Online transactions showed significantly higher fraud percentage compared to chip and swipe transactions.

2. High-Value Transactions Are Riskier
High-value transactions had the highest fraud rate despite lower transaction volume.

3. Card Type 0 Had Highest Fraud Risk
Certain card categories showed much higher fraud concentration than others.

4. Fraud Activity Peaks in October and November
Fraud percentage sharply increased during these months.

5. Afternoon Transactions Had Higher Fraud Rate
Most fraud activity occurred during afternoon transaction periods.
